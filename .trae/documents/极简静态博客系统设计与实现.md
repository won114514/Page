# 极简静态博客系统设计与实现

## 技术栈选择
- **构建工具**: Astro（内容优先、轻量、支持多种框架集成）
- **包管理**: pnpm
- **语言**: TypeScript
- **Markdown 支持**: Astro 内置 + Shiki 代码高亮
- **UI 框架**: 原生 HTML + CSS（极简设计）
- **响应式设计**: CSS Grid + Flexbox
- **深色/浅色模式**: CSS 变量 + 媒体查询

## 项目结构
```
my-blog/
├── content/                 # 文章内容目录
│   └── posts/              # Markdown 文章
│       ├── hello-world.md  # 示例文章1
│       └── my-thoughts.md  # 示例文章2
├── src/
│   ├── components/         # 可复用组件
│   │   ├── Header.astro    # 头部组件
│   │   ├── Footer.astro    # 底部组件
│   │   └── ThemeToggle.astro # 主题切换组件
│   ├── layouts/            # 布局组件
│   │   └── BaseLayout.astro # 基础布局
│   └── pages/              # 页面组件
│       ├── index.astro     # 首页（文章列表）
│       ├── [slug].astro    # 文章详情页
│       └── rss.xml.ts      # RSS 订阅
├── public/                 # 静态资源
│   └── favicon.ico
├── astro.config.mjs        # Astro 配置
├── tsconfig.json           # TypeScript 配置
├── package.json            # 依赖配置
└── README.md               # 项目说明
```

## 核心功能实现

### 1. Markdown 处理
- 使用 Astro 内置的 Markdown 支持
- 配置 Frontmatter 支持（YAML 格式）
- 集成 Shiki 实现代码高亮

### 2. 文章列表生成
- 从 `content/posts/` 目录读取所有 Markdown 文件
- 解析 Frontmatter 元数据
- 按日期倒序排序显示
- 支持分页（可选）

### 3. 文章详情页
- 动态路由 `[slug].astro` 匹配所有文章
- 显示文章内容、元数据
- 支持标签分类（可选）

### 4. 主题切换
- CSS 变量定义颜色方案
- 媒体查询自动适配系统主题
- 手动切换主题功能
- 主题偏好本地存储

### 5. RSS 订阅
- 生成标准 RSS 2.0 格式
- 包含所有文章标题、链接、摘要

## 依赖管理
- 尽量减少第三方依赖
- 仅使用必要的 Astro 插件
- 优先使用原生 Web API
- 所有依赖使用 TypeScript 类型定义

## 部署支持
- 配置 Vercel/Netlify/GitHub Pages 自动部署
- 构建输出到 `dist/` 目录
- 支持自定义域名

## 开发体验
- 热重载支持
- 类型检查
- 清晰的错误提示
- 快速构建速度

这个设计遵循了内容优先的原则，让用户只需专注于编写 Markdown 文章，无需关心样式和部署细节。同时保持了高可维护性和扩展性，便于后续功能增强。
